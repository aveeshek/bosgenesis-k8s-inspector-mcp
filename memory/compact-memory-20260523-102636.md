## Mission

Namespace-scoped Kubernetes MCP/REST server for BOS Genesis. Operates only inside `bosgenesis` through remote MCP `/mcp` and in-cluster ServiceAccount auth.

## Non-Negotiable Safety Rules

- Only `bosgenesis` namespace.
- Prefer MCP tools over raw `kubectl`.
- No cluster-admin.
- No Secret read/list/get/update/patch.
- No generic Secret apply/create/update/patch/delete.
- No RBAC, ServiceAccount, Node, Namespace, PV, or CRD mutation.
- No pod exec/attach/port-forward.
- Mutations require API key, policy validation, audit records, and dry-run first when possible.

## Key Design Decisions

- Remote MCP endpoint: `http://k8s-inspector.bosgenesis.local/mcp`
- Codex config should use remote URL, not local kubeconfig.
- ConfigMap list returns metadata/key names only.
- ConfigMap get returns values only with `include_data=true`.
- Ephemeral Secret workflow is write-only and separate from generic manifests.
- Ephemeral Secret names must start with `bosgenesis-mcp-`.
- Secret values are never returned.
- Ephemeral Secret delete requires in-session `correlation_id`.
- Secret TTL is in-memory plus annotation only; explicit delete is required.
