## Project Purpose

`bosgenesis-k8s-inspector-mcp` is a namespace-scoped Kubernetes MCP/REST server for BOS Genesis. It lets Codex and other agents inspect and operate allowed Kubernetes resources only inside the `bosgenesis` namespace.

## Hard Guardrails

- Only operate inside namespace `bosgenesis`.
- Do not use `kubectl` for cluster inspection when MCP tools are available.
- Do not request or use cluster-admin.
- Do not inspect or mutate Kubernetes Secrets.
- Do not create or modify RBAC, ServiceAccounts, Namespaces, Nodes, PersistentVolumes, or CRDs.
- Do not use pod exec, attach, or port-forward.
- All mutations go through policy validation.
- Prefer dry-run before real mutations.
- All MCP/API operations must emit audit records.

## ConfigMap Read Decision

ConfigMap direct read was added without weakening the Secret guardrail.

Design:

- `k8s_list_configmaps` lists ConfigMaps with metadata and key names only.
- `k8s_get_configmap` reads one ConfigMap.
- `k8s_get_configmap` returns metadata and key names by default.
- ConfigMap values are returned only when `include_data=true`.
- Secrets remain blocked by policy and RBAC.

REST endpoints added:

- `GET /configmaps`
- `GET /configmaps/{configmap_name}?include_data=false`
