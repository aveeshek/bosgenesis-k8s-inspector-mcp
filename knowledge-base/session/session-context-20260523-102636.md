## Purpose

`bosgenesis-k8s-inspector-mcp` is a namespace-scoped Kubernetes MCP/REST server for BOS Genesis. It lets Codex and other agents inspect and safely operate approved Kubernetes resources only inside the `bosgenesis` namespace.

## Hard Guardrails

- Only operate inside namespace `bosgenesis`.
- Do not use `kubectl` for cluster inspection when MCP tools are available.
- Do not request or use cluster-admin.
- Do not read, list, get, update, patch, or generically apply Kubernetes Secrets.
- Do not create or modify RBAC, ServiceAccounts, Namespaces, Nodes, PersistentVolumes, or CRDs.
- Do not use pod exec, attach, or port-forward.
- Mutations require API key and policy validation.
- Prefer dry-run before real mutation.
- Audit all operations.

## Current Features

Remote MCP:

- Endpoint: `http://k8s-inspector.bosgenesis.local/mcp`
- Uses in-cluster Kubernetes ServiceAccount auth.
- ServiceAccount bearer token is explicitly added to the Kubernetes `ApiClient` to avoid anonymous API calls.

ConfigMap tools:

- `k8s_list_configmaps`
- `k8s_get_configmap`
- REST:
  - `GET /configmaps`
  - `GET /configmaps/{configmap_name}?include_data=false`
- List returns metadata and key names only.
- Get returns values only when `include_data=true`.

Safe ephemeral Secret workflow:

- `k8s_create_ephemeral_secret`
- `k8s_delete_ephemeral_secret`
- REST:
  - `POST /secrets/ephemeral`
  - `POST /secrets/ephemeral/delete`
- Secret values are never returned in MCP, REST, audit summaries, or docs.
- Secret names must start with `bosgenesis-mcp-`.
- Deletion requires matching in-session `correlation_id`.
- TTL is stored in process memory and as an annotation; Kubernetes does not auto-delete Secrets from custom annotations, so callers must explicitly delete temporary Secrets.
- RBAC grants only `create` and `delete` on `secrets`; it does not grant `get`, `list`, `watch`, `update`, or `patch`.
